3-4. 構成図の読み方・描き方

フクロウ先生より

提案書やレビュー会議で必ず出てくるのがシステム構成図です。「図が読めない＝会議についていけない」ということ。でも心配いりません！アイコンの意味と5つの確認観点を覚えれば、構成図はぐっと身近になりますよ。

重要ポイント

構成図で大切なのは「矢印の方向」です。矢印は「データの流れ」や「リクエストの方向」を示します。箱（コンポーネント）の名前よりも、どこからどこにデータが流れるかを追うと理解しやすくなります。

最も基本的な構成図です。多くの業務システムはこの形がベースになっています。

営業が確認すべきポイント

  • Webサーバーが2台 → 冗長化されている（1台止まってもサービス継続）
  • DB ReplicaがPrimaryと別にある → 読み取り負荷分散＆バックアップ
  • CDNがある → 静的ファイルの配信が高速、サーバー負荷軽減

開発チームの視点

この構成では各サービスが「自分専用のDB」を持っています。これはDatabase per Serviceパターンと呼ばれ、マイクロサービスの基本原則です。営業が「なぜDBが4つもあるのか？」と疑問に思ったときの答えは「サービス間の独立性を保つため」です。

営業トーク例

構成図を読む「5つの確認観点」

構成図を見たとき、以下の5つの観点で確認すると、的確な質問ができるようになります。提案書レビューや技術レビュー会議で即使えるチェックリストです。

  1. 冗長化：単一障害点（SPOF）はないか？ 各コンポーネントが1つしかない場合、そこが止まるとシステム全体が止まります。「この箱が1台だけですが、止まった場合の影響は？」と確認しましょう。
  2. データの流れ：矢印の方向は正しいか？ リクエストはどこから始まり、どこまで到達するか。逆方向のレスポンスは省略されていることが多いですが、双方向通信（WebSocket等）が必要な場合は要確認です。
  3. セキュリティ境界：ネットワーク区分は適切か？ インターネットからアクセスできる範囲（パブリック）と、内部だけの範囲（プライベート）は明確ですか？「DBがインターネットから直接アクセスできる構成になっていませんか？」は重要な確認ポイントです。
  4. スケールポイント：ボトルネックはどこか？ 負荷が集中するポイントはどこか、そこがスケールアウト可能な構成になっているかを確認します。「アクセスが10倍になった場合、この構成で耐えられますか？」と聞きましょう。
  5. 外部連携：外部システムとの接続方式は？ 外部API・レガシーシステムとの接続部分は障害の影響を受けやすいポイントです。「外部決済APIが停止した場合、注文処理はどうなりますか？」と確認しましょう。

豆知識

構成図を「描く」必要はなくても、「読んで質問できる」だけで大きな差がつきます。レビュー会議で上の5つの観点から1つでも質問できると、「この営業さん、ちゃんとわかっている」と技術チームから信頼されます。

実践：構成図にツッコミを入れてみよう

以下の構成図にはいくつかの「気になるポイント」があります。5つの確認観点を使って問題点を見つけてみましょう。

あるECサイトの構成図（ツッコミ対象）
👤ユーザー
🖥️Webサーバー
（1台のみ）
⚙️APサーバー
🔗決済API
（障害時は？）
🗄️DB
（1台のみ）

解答：見つけるべきポイント

  • 観点1（冗長化）：Webサーバーが1台 → SPOF。障害時にサービス全停止
  • 観点1（冗長化）：DBも1台 → SPOF。データ損失のリスクもある
  • 観点3（セキュリティ）：ネットワーク境界が描かれていない → パブリック/プライベートの区分が不明
  • 観点5（外部連携）：決済API障害時のフォールバック（代替手段）が不明
  • 観点4（スケール）：ロードバランサーがない → 水平スケール不可

確認クイズ

Q1. 構成図で「SPOF（単一障害点）」を見つけるには、どこに注目すべき？

SPOFは「1つしかないコンポーネント」です。そこが止まるとシステム全体が止まります。冗長化（2台以上の構成）がされているかを確認しましょう。

Q2. マイクロサービス構成でサービスごとにDBを分ける理由は？

Database per Serviceパターンは、各サービスが自分のデータだけに責任を持ち、他サービスに影響を与えずに独立して開発・デプロイできるようにするためです。

Q3. 構成図を読む5つの確認観点に含まれないものはどれ？

構成図から読み取れるのはコンポーネントの配置と通信の流れです。開発言語の選定は構成図の範疇ではなく、別途確認すべき事項です。5つの観点は「冗長化・データの流れ・セキュリティ境界・スケールポイント・外部連携」です。
