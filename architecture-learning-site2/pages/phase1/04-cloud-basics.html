🌱1-4. クラウドの基礎

フクロウ先生の導入

「うちもそろそろクラウドに移行したい」——お客様からこう言われたこと、ありませんか？でも「クラウド」と一言で言っても、その中身はさまざま。今日は、クラウドの基礎をレストランのたとえで楽しく学んでいきましょう。食事をするにも「外食」「ミールキット」「食材を買って自炊」と選択肢がありますよね。クラウドにも同じような選択肢があるんです！

SaaS / PaaS / IaaS — レストランのたとえ

クラウドサービスは、「どこまで自分でやるか」によって3つに分類されます。

分類 レストランのたとえ 自分でやること お任せする部分 代表例
SaaSSoftware as a Service。完成したソフトウェアをインターネット経由で利用するサービス形態 外食（レストランで食べる） 食べるだけ（使うだけ） 調理、食材、キッチン、建物すべて Microsoft 365、Salesforce、Google Workspace
PaaSPlatform as a Service。アプリを動かす基盤（OS、ミドルウェア等）をクラウドで提供するサービス形態 ミールキット（キッチン付き貸しスペース） 調理（アプリ開発） キッチン設備、建物（OS、ミドルウェア、インフラ） AWS Elastic Beanstalk、Azure App Service、Heroku
IaaSInfrastructure as a Service。サーバー、ネットワーク、ストレージなどのインフラをクラウドで提供するサービス形態 食材を買って自炊（キッチンだけ借りる） 調理＋キッチン管理（アプリ＋OS設定） 建物（サーバー機器、ネットワーク物理設備） AWS EC2、Azure Virtual Machines、GCE

ポイント

選ぶ基準は「自由度」と「手間」のトレードオフです。SaaSは手間ゼロだけどカスタマイズの自由度は低い。IaaSは自由度が高いけど運用の手間がかかる。お客様の「やりたいこと」と「運用できる体制」を聞けば、どれが合うか見えてきます。

3大クラウドベンダーの特徴

クラウドの話をするとき、必ず出てくるのがAWS、Azure、GCPの3大ベンダーです。営業的に押さえておきたいポイントを整理しましょう。

ベンダー シェア 強み 営業的な勘所
AWS（Amazon） 世界シェア1位（約30%） サービスの種類が最多、エンジニア人材が豊富、事例が多い 「迷ったらAWS」と言われるほどの安定感。お客様のIT部門にAWS経験者がいることが多い
Azure（Microsoft） 世界シェア2位（約25%） Microsoft製品との親和性、エンタープライズ実績、Active Directory連携 お客様がMicrosoft 365を使っているなら有力候補。日本の大企業での採用が増加中
GCP（Google） 世界シェア3位（約10%） データ分析・AI/ML、BigQueryの圧倒的性能、コンテナ技術 データ活用やAIを重視するお客様に刺さりやすい。分析基盤の案件では有力候補

豆知識

日本市場に限ると、AWSとAzureがほぼ拮抗しています。お客様がすでに使っているクラウドがあるなら、まずはそれに合わせるのがスムーズ。複数のクラウドを併用するマルチクラウド複数のクラウドベンダーのサービスを組み合わせて利用する戦略。ベンダーロックイン対策になる戦略を取る企業も増えています。

オンプレミス vs クラウド vs ハイブリッド

クラウド以前は、自社でサーバーを持つオンプレミス自社の建物内にサーバーやネットワーク機器を設置して運用する方式。「自社設置型」とも呼ばれるが主流でした。今でもオンプレミスには利点があり、多くの企業は両方を組み合わせたハイブリッドクラウドオンプレミスとクラウドを組み合わせて利用する構成。それぞれの利点を活かせるを採用しています。

比較項目 オンプレミス クラウド ハイブリッド
初期費用 高い（サーバー購入） 低い（従量課金） 中程度
運用費用 人件費＋電気代＋保守費 利用料（使った分） 両方のコストが発生
拡張性 機器の追加購入が必要（数週間〜） 数分で拡張可能 クラウド側で柔軟に対応
セキュリティ 自社で完全管理（物理的に隔離） ベンダーの対策＋自社の設定管理 用途に応じて使い分け可能
適したケース 法規制で外部保管不可、超低遅延が必要 変動するワークロード、グローバル展開 段階的なクラウド移行、特定データの社内保管

お客様が「クラウドに移行したい」と言ったら

この一言を聞いたとき、すぐに見積もりを出すのではなく、まず以下を確認しましょう。

  1. 「何を」移行したいのか？——全部？一部？新しいシステムだけ？
  2. 「なぜ」移行したいのか？——コスト削減？拡張性？BCP対策？
  3. 「いつまでに」移行したいのか？——既存システムの保守期限は？
  4. 現在のインフラ状況は？——オンプレのサーバーは何台？OSは？
  5. データの制約は？——個人情報や機密データをクラウドに置けるか？法規制は？

ポイント

お客様の「クラウドに移行したい」は、多くの場合「コストを下げたい」「運用を楽にしたい」が本音です。クラウド移行そのものが目的ではなく手段であることを念頭に、「何を解決したいのか」を引き出すのが営業の腕の見せどころです。

営業の現場では...

お客様がクラウドに不安を感じている場合、「セキュリティが心配」「データが外に出るのは...」という声をよく聞きます。こんなとき、「実はメガバンクもクラウドを採用しています。クラウドベンダーのセキュリティ対策は、一般企業の自社対策を上回ることも多いんです」と具体的な事例で説明できると安心感を与えられます。ただし、業界の法規制（金融庁のガイドラインなど）は必ず確認しましょう。

開発チームの視点

開発チームにとって、クラウド選定で重要なのは「既存スキル」と「エコシステム」です。チームにAWS経験者が多いのにGCPを選ぶと学習コストがかかります。また、「クラウドにすればコストが下がる」とは限りません。常時稼働する基幹システムはオンプレの方が安いこともあります。営業さんにお願いしたいのは、お客様のIT部門が今何を使っているかを聞いてくること。これだけで技術選定がかなりスムーズになります。

理解度チェック

Q1

SaaS / PaaS / IaaS

レストランのたとえで、「キッチン付き貸しスペースで自分で調理する」に相当するクラウドサービスはどれですか？

  • A SaaS
  • B PaaS
  • C IaaS
  • D オンプレミス

PaaSは「キッチン付き貸しスペース（ミールキット）」にあたります。OS やミドルウェアなどの基盤（キッチン設備）はクラウドベンダーが用意し、利用者はアプリケーションの開発（調理）に集中できます。SaaSは外食（使うだけ）、IaaSは食材を買って自炊（インフラから自分で管理）です。

Q2

クラウドベンダーの選び方

お客様が社内でMicrosoft 365を全社導入しています。クラウド基盤として最も親和性が高いのはどれですか？

  • A AWS（AWSの方がシェアが高いから）
  • B GCP（Googleのサービスだから）
  • C Oracle Cloud（データベースが強いから）
  • D Azure（Microsoft製品との連携が強みだから）

Microsoft 365を導入済みのお客様には、同じMicrosoftのAzureが最も親和性が高くなります。Active Directory（ユーザー管理）の連携やシングルサインオンがスムーズに実現できるためです。もちろん他のクラウドでも連携は可能ですが、既存のIT環境との親和性はベンダー選定の重要なポイントです。

Q3

クラウド移行のヒアリング

お客様が「クラウドに移行したい」と言ったとき、最初に確認すべきことはどれですか？

  • A AWSとAzureのどちらにするか決めてもらう
  • B すぐに移行スケジュールを提案する
  • C 移行の目的（なぜ？）と対象範囲（何を？）を確認する
  • D クラウドの料金表を見せる

ベンダーやスケジュールの話の前に、まず「なぜ移行したいのか」「何を移行したいのか」を確認することが大切です。コスト削減が目的なのか、拡張性が目的なのかで最適な方法はまったく変わります。目的と範囲が明確になれば、SEへの引き継ぎもスムーズに進みますよ。
